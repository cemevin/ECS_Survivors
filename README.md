# ECS Survivors

A Vampire Survivors-style demo built in a weekend to get up to speed with 
Unity 6 ECS/DOTS as a gameplay programmer. Focuses on performance 
characteristics and developer ergonomics of the DOTS stack.

## Video Demonstration

[![Watch the video](https://img.youtube.com/vi/wdxXGaRUfCA/maxresdefault.jpg)](https://www.youtube.com/watch?v=wdxXGaRUfCA)

## Features

- 1000+ simultaneous enemies via Unity ECS
- Burst-compiled systems — movement, collision, spawning, shockwave, animation
- Spatial hash broadphase collision
- Sprite sheet animation via `[MaterialProperty]` per-entity shader overrides
- Shader-driven health bars with dirty flag event-driven updates
- Shockwave mechanic with ring expansion shader and enemy impulse
- Hybrid ECS/MonoBehaviour for player visuals, camera, and UI Toolkit HUD
- Runtime spawner settings menu

## Architecture
```
Components/     — pure IComponentData structs
Systems/        — ISystem (Burst) and SystemBase (managed) 
Authoring/      — MonoBehaviour bakers for SubScene baking
Bootstrap/      — app settings, VFX manager
```

## Key Technical Decisions

**Spatial hashing** — `NativeParallelMultiHashMap` buckets enemies by grid cell, 
reducing projectile collision from O(projectiles × enemies) to O(projectiles × 
enemies per cell).

**IEnableableComponent** — used for `DyingTag` and `ShockwaveImpactCooldown` 
to avoid archetype changes on state transitions, preventing chunk fragmentation 
on mass death events.

**ECBSystem** — all structural changes go through `BeginSimulation` or 
`EndSimulationEntityCommandBufferSystem` to avoid mid-frame sync points.

**Dirty flags** — health bar updates are event-driven via `HealthDirty` 
`IEnableableComponent`, only processing entities that took damage that frame.

**Hybrid approach** — player visuals, camera, and UI use MonoBehaviours that 
read from the ECS world each frame. Avoids forcing ECS onto systems that don't 
benefit from it.

## Known Limitations / ECS Pain Points

- Parent/child entity hierarchy doesn't propagate automatically in all cases
- No enforced contract between systems and components — stale queries fail silently
- Authoring/baker/entity separation adds significant boilerplate
- Managed APIs (Camera, Input) can't be Burst compiled — requires hybrid bridging
